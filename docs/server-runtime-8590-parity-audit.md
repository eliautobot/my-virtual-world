# Server Runtime ↔ 8590 Parity Audit

**Goal:** the server-authoritative Virtual World (Colyseus sidecar, observed on staging + realtime :8588) must feel exactly like the stable browser-owned build on 8590. Server owns all behavior; browsers are observers.

**Scope of this audit:** behavior parity, not architecture. Architecture (server ownership, leases, single-instance gating, queues, collision, manual placement) landed in PR #48–#57. This doc quantifies the *behavior richness* gap and points to root causes for PR #58.

### Headline numbers

- **8590 browser obs (26 agents):** 11 distinct animation ids: `stand-use(199) bus-stop-wait(210) gather-talk(116) write-teach(95) carry-right-hand(76) vending-machine-use(26) sit(18) inspect-browse(17) microwave-use(10) printer-scanner-use(9) run-in-place(4)`.
- **staging browser obs (25 agents):** 19 distinct animation ids (incl. `typing`, `meeting-sit-talk`, `play-pingpong`, `water-desk-sip`, `order-food-drink`) — the object-use *animation* vocabulary is actually competitive.
- **staging server samples:** 18 distinct activity kinds; state mix `idle 1100 / working 798 / routing 641 / using 437 / waiting 132 / in_progress 12` (per-agent-sample counts).
- **Critical absences in staging:** `gather-talk` **never appears** (116 occurrences on 8590 → conversations/social system missing entirely), and one agent (`permi`) spent **120/120 samples in `routing`** (stuck ~107 min).
- Vocabulary superset check: 8590 code contains ~1,652 distinct kebab-case string kinds vs ~346 in the server runtime; most missing ones are the social/role/expression layers, not object-use kinds (the server's `SERVER_SCRIPTED_OBJECT_ACTIVITY_CONFIG` at `agent-runtime-room.mjs:2398` already covers ~100 object types).

## Stuck-agent case detail: `permi`

- 120/120 server samples in `state: routing`, `owner: server-scripted-object-runtime`, activity `service-queue-wait`, target `armchair` queue slot `approach-front:0` in `live-home-adam`.
- `target.runtimeStartedAt = 2026-07-01T23:16:02Z`; still routing at `updatedAt = 2026-07-02T01:03:02Z` → **>105 minutes** on one route.
- Target coords `(x:1492, y:-1036)` vs agent position `(≈500, -28)` orbiting within a ~10-unit blob near the building. Strong signal the queue-slot target was emitted in a different coordinate frame (world-scaled/interior grid) than the movement integrator consumes, so arrival radius (5 units) is never satisfiable.
- 8590 equivalent guard: agent-intent approach staleness at **45s** (`AGENT_INTENT_APPROACH_STALE_AFTER_MS`, main3d.js:7387) would have aborted this 140× over.
- Also note `stayMs: 11705` on the target — even the intended dwell was 12s; the route itself is the hang.

## Corrections to prior agents' findings

1. **Facial expressions:** 8590 `main3d.js` contains zero matches for "expression"; per-activity *animation* variety is real, facial expressions are not a 8590 feature. Dropped to non-goal.
2. **Walk speed:** scripted-object runtime already matches (72 vs 70); the real offender is the **live-status runtime walk speed 96**.
3. **Object vocabulary:** the server config map is far more complete than assumed (~100 types). The genuine gaps are (a) social variants / partner recruitment, (b) worker+customer service roles, (c) a few exterior types (fountain, gazebo, food-truck counter — verify keys), (d) elevator transitions.
